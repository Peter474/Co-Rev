Chapter 1: Introduction to Computer Organization

Basic Information Representation

Computers are digital devices that work with discrete electrical signals:

  • High Level (1): Typically 5V or 12V
  • Low Level (0): Typically 0V
Binary Number System: Uses only 0 and 1 to represent all information
  • Bit: Binary Digit (0 or 1)
  • Nibble: 4 bits
  • Byte: 8 bits

Computer Architecture vs Organization

Computer Architecture Computer Organization
Describes what the computer does Describes how it does it
Functional behavior of systems Structural relationships
High-level design (instruction sets, registers, data types) Low-level design (circuits, peripherals, control signals)
Fixed first in design process Decided after architecture
Also called Instruction Set Architecture Also called Microarchitecture

Von Neumann Architecture

Modern computers are based on three key concepts:

  1. Data and instructions stored in single read-write memory
  2. Memory contents addressable by location, regardless of data type
  3. Execution occurs sequentially unless explicitly modified

Four Main Components

CPU (Central Processing Unit)

  • Control Unit
  • Arithmetic Logic Unit (ALU)
  • Registers

Main Memory

  • Volatile: RAM
  • Non-volatile: ROM, PROM, EPROM

I/O Devices

  • Input: Keyboard, Mouse
  • Output: Monitor, Printer

System Interconnection

  • System Bus
  • Data lines
  • Address lines
  • Control lines

Basic Computer Functions

  • Data Processing: Arithmetic and logical operations
  • Data Storage: Temporary and permanent storage
  • Data Movement: Input/Output operations
  • Control: Managing the sequence of operations

Chapter 3: Computer Function & Interconnection

Basic Instruction Cycle

Two main phases:

  1. Fetch Cycle: Retrieve instruction from memory
  2. Execute Cycle: Perform the operation
Key Registers:
  • PC (Program Counter): Address of next instruction
  • IR (Instruction Register): Holds current instruction
  • AC (Accumulator): Stores data temporarily
  • MAR (Memory Address Register): Address for memory access
  • MBR (Memory Buffer Register): Data to/from memory

Instruction Cycle States

  • IAC: Instruction Address Calculation
  • IF: Instruction Fetch
  • IOD: Instruction Operation Decoding
  • OAC: Operand Address Calculation
  • OF: Operand Fetch
  • DO: Data Operation
  • OS: Operand Store

Interrupts

Purpose: Improve processing efficiency by allowing processor to handle multiple tasks

Classes of Interrupts:

  • Program: Arithmetic overflow, division by zero
  • Timer: Generated by internal timer
  • I/O: From I/O controller
  • Hardware failure: Power failure, memory error
Multiple Interrupts Handling:
  • Disable interrupts: Process one at a time
  • Priority system: Higher priority can interrupt lower priority handlers

Bus Interconnection

A bus is a shared communication pathway connecting multiple devices.

Three Types of Bus Lines:

Data Bus

Carries actual data between components. Width (32, 64, 128 bits) determines performance.

Address Bus

Specifies memory location or I/O port. Width determines max memory capacity.

Control Bus

Carries control signals: Memory Read/Write, I/O Read/Write, Bus Request/Grant, Clock, Reset

Point-to-Point Interconnection

Modern systems use point-to-point connections instead of shared buses:

  • QPI (Quick Path Interconnect): Intel's technology
  • PCIe (PCI Express): High-speed peripheral connections
Advantages:
  • Lower latency
  • Higher data rates
  • Better scalability
  • Multiple direct connections between components

Chapter 4: Cache Memory

Memory Hierarchy

Organized from fastest/smallest to slowest/largest:

  1. Registers - Inside CPU, fastest
  2. Cache (L1, L2, L3) - SRAM, very fast
  3. Main Memory - DRAM, moderate speed
  4. Secondary Storage - HDD/SSD, slowest
Trade-offs:
  • Faster access time = Greater cost per bit
  • Greater capacity = Smaller cost per bit
  • Greater capacity = Slower access time

Cache Memory Principles

Cache stores frequently accessed data from main memory for faster retrieval.

Key Performance Metrics:

  • Hit Ratio: Percentage of memory accesses found in cache
  • Miss Penalty: Time cost when data not in cache
Locality Principle:
  • Temporal: Recently accessed data likely accessed again
  • Spatial: Data near recently accessed data likely accessed

Cache Mapping Functions

1. Direct Mapping

Each memory block maps to exactly one cache line

  • Formula: i = j mod m (i=cache line, j=block number, m=cache lines)
  • Advantage: Simple, fast
  • Disadvantage: Thrashing if two blocks compete for same line

2. Fully Associative Mapping

Any memory block can go in any cache line

  • Advantage: Most flexible, best hit ratio
  • Disadvantage: Complex, expensive, slower

3. Set-Associative Mapping

Compromise between direct and fully associative (k-way set associative)

  • Advantage: Balance of performance and complexity
  • Common: 2-way, 4-way, 8-way associative

Replacement Algorithms

Algorithm Description Characteristics
LRU (Least Recently Used) Replace block unused longest Most effective, most popular
FIFO (First In First Out) Replace oldest block Simple round-robin
LFU (Least Frequently Used) Replace least accessed block Uses counters
Random Replace random block Simplest implementation

Write Policies

Write Through

  • Write to both cache and memory simultaneously
  • Advantage: Memory always up-to-date
  • Disadvantage: More memory traffic

Write Back

  • Write only to cache, update memory when block replaced
  • Advantage: Fewer memory writes
  • Disadvantage: Complex, memory can be outdated

Multilevel Caches

  • L1 Cache: Smallest, fastest, on-chip
  • L2 Cache: Larger, slower than L1
  • L3 Cache: Largest, shared among cores
Split vs Unified Cache:
  • Split (L1): Separate instruction and data caches
  • Unified (L2/L3): Combined cache for both

Chapter 5: Internal Memory

Memory Types Overview

RAM (Random Access Memory)

DRAM (Dynamic RAM)
  • Stores data as charge on capacitors
  • Requires periodic refresh
  • Simpler, smaller, cheaper
  • Used for main memory
SRAM (Static RAM)
  • Uses flip-flop circuits
  • No refresh needed
  • Faster but more expensive
  • Used for cache memory
Both are Volatile: Data lost when power removed

ROM (Read-Only Memory)

Type Write Capability Erase Method
ROM Factory programmed only Cannot erase
PROM Write once Cannot erase
EPROM Multiple writes UV light
EEPROM Multiple writes (byte-level) Electrical
Flash Memory Multiple writes (block-level) Electrical

Error Correction

Types of Errors:

  • Hard Failure: Permanent physical defect
  • Soft Error: Random, non-destructive event (power issues, alpha particles)

Hamming Code

Error-correcting code that can detect and correct single-bit errors

  • SEC (Single Error Correcting): Corrects 1-bit errors
  • SEC-DED: Corrects 1-bit, detects 2-bit errors
For M-bit data word, need K check bits where: 2^K ‚â• M + K + 1

Advanced DRAM Organizations

SDRAM (Synchronous DRAM)

  • Synchronized with system clock
  • No wait states
  • Pipelined operations

DDR SDRAM (Double Data Rate)

  • Transfers data on both clock edges
  • DDR, DDR2, DDR3, DDR4, DDR5 generations
  • Each generation doubles effective speed

RDRAM (Rambus DRAM)

  • Proprietary technology
  • High bandwidth bus
  • Used in some Pentium systems

Memory Organization

Interleaved Memory

Multiple memory banks that can service requests simultaneously

  • K banks = K times faster for sequential access
  • Consecutive words stored in different banks
  • Improves throughput significantly

Chapter 6: External Memory

Magnetic Disk Technology

Physical Structure:

  • Platter: Circular disk coated with magnetic material
  • Track: Concentric circles on platter surface
  • Sector: Segments of a track
  • Cylinder: Same track across all platters
  • Read/Write Head: Magnetizes/reads surface

Disk Classifications:

  • Fixed vs Movable Head: One head per track vs moving head
  • Removable vs Non-removable: Can be swapped or permanent
  • Winchester Disk: Sealed unit, heads fly on air cushion

Disk Performance Parameters

  • Seek Time: Time to position head at desired track
  • Rotational Latency: Time for sector to reach head
  • Access Time: Seek time + Rotational latency
  • Transfer Time: Time to read/write data

Total Time = Seek + Rotation + Transfer

RAID (Redundant Array of Independent Disks)

Combines multiple disks for performance and/or reliability

RAID Level Description Min Disks Redundancy
RAID 0 Striping, no redundancy 2 None
RAID 1 Mirroring 2 Complete copy
RAID 2 Bit-level striping with Hamming code 3+ Error correction
RAID 3 Byte-level striping with parity 3 Single parity disk
RAID 4 Block-level striping with parity 3 Single parity disk
RAID 5 Block striping, distributed parity 3 Distributed parity
RAID 6 Block striping, double parity 4 Two parity blocks
Most Common:
  • RAID 0: Maximum performance, no protection
  • RAID 1: Simple mirroring, 50% capacity
  • RAID 5: Good balance of performance, capacity, and protection
  • RAID 10 (1+0): Mirrored stripes, excellent performance and protection

Solid State Drives (SSD)

Flash Memory Types:

NOR Flash
  • Bit-level access
  • Random access
  • Used for BIOS, OS code
NAND Flash
  • Block-level access (16-32 bits)
  • Higher density
  • Used in USB drives, SSDs, memory cards

SSD Advantages over HDD:

  • Higher IOPS (Input/Output Operations Per Second)
  • Lower access times and latency
  • More durable (no moving parts)
  • Lower power consumption
  • Quieter and cooler operation
  • Longer lifespan
SSD Limitations:
  • Performance degrades over time
  • Limited write cycles per cell
  • Entire block must be erased before writing
Solutions: Wear leveling, bad-block management, over-provisioning

Optical Storage

CD-ROM (Compact Disc Read-Only Memory)

  • Data stored as microscopic pits on polycarbonate
  • Read by laser reflection
  • Capacity: ~700 MB
  • Read-only, mass production

CD-R (Recordable)

  • Write-once, read-many
  • Uses dye layer changed by laser

CD-RW (Rewritable)

  • Phase-change technology
  • Can be rewritten multiple times

DVD (Digital Versatile Disk)

  • Higher density than CD
  • Single layer: 4.7 GB
  • Dual layer: 8.5 GB

Blu-ray

  • Blue laser (shorter wavelength)
  • Single layer: 25 GB
  • Dual layer: 50 GB

Magnetic Tape

  • Sequential access medium
  • High capacity, low cost
  • Used for backup and archival
  • LTO (Linear Tape-Open): Industry standard

Chapter 8: Operating System Support

Operating System Overview

An OS is a program that controls application execution and acts as an interface between applications and hardware.

Two Main Objectives:

  1. Convenience: Makes computer easier to use
  2. Efficiency: Allows efficient use of system resources

OS Services:

  • Program creation (utilities)
  • Program execution
  • Access to I/O devices
  • Controlled file access
  • System access control
  • Error detection and response
  • Accounting

Types of Operating Systems

Batch System

  • Jobs batched together
  • No user interaction during execution
  • Maximizes throughput

Interactive System

  • User interacts directly
  • Immediate feedback
  • Better user experience

Multiprogramming

  • Multiple programs in memory
  • CPU switches between them
  • Better CPU utilization

Time Sharing

  • Multiple users share CPU time
  • Small time quantum for each
  • Appears simultaneous to users

Process Scheduling

Scheduler Type Responsibility Frequency
Long-Term Job admission to system Minutes/hours
Medium-Term Swapping decisions Seconds/minutes
Short-Term CPU allocation (dispatcher) Milliseconds
I/O I/O device allocation Variable

Five-State Process Model

  • New: Process being created
  • Ready: Waiting for CPU
  • Running: Executing on CPU
  • Blocked/Waiting: Waiting for I/O or event
  • Exit: Process completed or terminated
Process Control Block (PCB) contains:
  • Process ID
  • Process state
  • Program counter
  • CPU registers
  • Memory management info
  • I/O status
  • Accounting information

Memory Management

Swapping

Moving processes between main memory and disk to manage memory efficiently

Partitioning

Fixed Partitioning
  • Memory divided into fixed-size regions
  • Simple but wasteful (internal fragmentation)
Dynamic Partitioning
  • Partitions created as needed
  • Exact size for each process
  • External fragmentation problem

Paging

  • Memory divided into fixed-size frames
  • Processes divided into fixed-size pages
  • Pages can be non-contiguous in memory
  • No external fragmentation
  • Uses page table for address translation

Segmentation

  • Memory divided into variable-size segments
  • Each segment is a logical unit (code, data, stack)
  • Visible to programmer
  • Easier to share and protect

Virtual Memory

Allows execution of processes that may not be completely in memory

Key Concepts:

  • Demand Paging: Pages loaded only when needed
  • Page Fault: Reference to page not in memory
  • Page Replacement: Select victim page when memory full
  • Working Set: Pages currently in use by process
Thrashing: System spends more time paging than executing
Caused by too many processes or poor page replacement

Page Replacement Algorithms:

  • FIFO: Replace oldest page
  • LRU: Replace least recently used page
  • Optimal: Replace page not used for longest time (theoretical)
  • Clock: Approximation of LRU using reference bit

Translation Lookaside Buffer (TLB)

Hardware cache for page table entries

  • Speeds up virtual-to-physical address translation
  • Contains recently used page table entries
  • Parallel search (associative memory)
  • Typical size: 32-1024 entries
Address Translation Steps:
  1. Check TLB for page number
  2. If TLB hit, get frame number immediately
  3. If TLB miss, access page table in memory
  4. Update TLB with new entry

Intel Pentium Memory Management

Four Memory Management Modes:

  1. Unsegmented Unpaged: Physical = Virtual address
  2. Unsegmented Paged: Linear address space with paging
  3. Segmented Unpaged: Logical segments without paging
  4. Segmented Paged: Both segmentation and paging

Key Features:

  • 16-bit segment selector + 32-bit offset
  • Two-level page tables
  • 4 KB or 4 MB page sizes
  • 4 privilege levels (0-3)
  • TLB with 32-128 entries

ARM Memory Management

Memory Region Types:

  • Supersections: 16 MB blocks
  • Sections: 1 MB blocks
  • Large Pages: 64 KB blocks
  • Small Pages: 4 KB blocks

Access Control:

  • 16 domains supported
  • Each region: No access, Read-only, or Read-write
  • Privileged vs user access
  • Clients: Respect access permissions
  • Managers: Bypass access permissions
